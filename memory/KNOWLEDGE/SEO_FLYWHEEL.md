# SEO 飞轮建议

> 来源：2026-03-29 闲聊 session 讨论 + 竞品调研
> 核心问题：目前用户来了→看了→走了，没有增长闭环，纯靠 SEO 单腿走路

---

## 竞品飞轮分析

| 竞品 | 飞轮机制 | 效果 |
|------|---------|------|
| Futurepedia | ① 徽章换反链（工具方放「Featured by Futurepedia」徽章在官网）② 开发者自助提交工具 | 月活 100万+，15万注册用户 |
| There's An AI For That | 任务导向搜索（「我要做XX」→推荐→每个任务自动成为 SEO 页） | 每个用户搜索 = 一个长尾页面 |
| Toolify.ai | 自动爬取新工具 + 流量排名 + 工具间对比页 | 持续自动产生新内容 |

**共同点：都有「别人帮我生产内容」的机制，不是纯靠自己写。**

## 我们缺的 + 建议方案（按优先级）

### 1️⃣ Stack 分享页（最优先，飞轮效应最强）

**机制：**
- 用户用 AI Builder 生成 stack → 保存 → 生成唯一 URL（如 `/stack/abc123`）
- 用户分享到 Twitter/LinkedIn/团队 → 带流量回来
- 每个 stack 页面 = 一个可被 Google 索引的 SEO 页面
- stack 越多 → SEO 页面越多 → 搜索流量越大 → 新用户创造新 stack → 飞轮

**工程量：** 小（数据库加 stacks 表 + 分享页路由 + OG 图）

**飞轮公式：**
```
用户 → AI 生成 stack → 保存 → 分享 URL → 社交传播 → 新用户 → 新 stack → ...
                                                    ↓
                                              Google 索引 → SEO 流量
```

### 2️⃣ Tool A vs Tool B 对比页（Programmatic SEO）

**机制：**
- 自动生成「n8n vs Zapier」「LangChain vs LlamaIndex」等对比页
- 120 个工具 × 120 = 理论 14,400 个长尾页面
- 实际取同类别工具对比 ≈ 500-1000 个高质量对比页
- 用 LLM 批量生成对比内容（成本极低）

**工程量：** 中（批量生成脚本 + 对比页模板 + 路由）

**搜索量验证：** 「X vs Y」类搜索在 AI 工具领域搜索量很大（用户做采购决策时常搜）

### 3️⃣ 工具提交表单（让别人帮你生产内容）

**机制：**
- 开放 `/submit` 页面，开发者自己提交工具信息
- 审核后上线（或自动上线 + 人工后审）
- 开发者有动力提交：被收录 = 免费曝光

**工程量：** 小（表单 + 审核队列）

### 4️⃣ 徽章/嵌入组件（换反向链接）

**机制：**
- 给被收录的工具提供嵌入代码：`<a href="agentoolrank.com/tool/xxx">Featured on AgentoolRank</a>`
- 工具方放在自己官网 → 反向链接 → 提升 SEO 域名权重
- 类似 Product Hunt 的「Featured on Product Hunt」徽章

**工程量：** 极小（生成一段 HTML 代码即可）

### 5️⃣ 周报 Newsletter

**机制：**
- 每周自动生成「本周新增工具 + 热门 Stack + AI 行业趋势」
- 用 LLM 自动写，成本 ≈ $0.01/期
- 邮件订阅 → 用户定期回访

**工程量：** 中（邮件系统 + 自动生成 + 订阅管理）

## AI Builder 输出升级（配合飞轮）

当前 AI 输出：工具列表 + JSON → 技术人看得懂，普通人不行

建议增加：**工作流故事**
- 每步标注用了哪个工具（【工具名】格式）
- 写成非技术人员能理解的用户场景描述
- 成本增加 ≈ $0.0003/次（可忽略）
- prompt 里加一段输出要求即可，DeepSeek 完全胜任

## 成本参考

- AI Builder 单次对话：$0.003（DeepSeek）
- 月 1万次对话：$30
- 对比页批量生成（1000页）：$5-10
- Newsletter 自动生成：$0.01/期
- **结论：AI 成本可以忽略，变现靠 affiliate 转化而非 AI 使用费**

## 数据壁垒（比 SEO 飞轮更重要）

> SEO 飞轮别人能抄，数据壁垒抄不走。

### 数据沉淀链路

1. AI 推荐 stack（每个环节推荐 2-3 个工具）
2. 用户逐个环节试用/注册（点击追踪）
3. 用户最终确认选哪个（决策数据）
4. 固化成「已验证 stack」

### 沉淀的独有数据

- 工具间真实搭配关系图谱（用户投票的，不是 AI 猜的）
- 各环节工具胜出率（「做 RAG 的人 70% 选了 Pinecone」）
- 任务→stack 的最优映射（越用越准）

### 数据壁垒的价值

- 推荐越来越准（数据飞轮）
- 别人抄不走（独有用户决策数据）
- 对工具方有议价权（用户在你和竞品之间选了谁）
- To Agent 接口：Agent 调 API 拿「真实用户验证过的 stack」
- 版权可防护：「基于 X 万用户决策的推荐」不是公开数据

### To Agent 方向

- 未来不是人来搜，是 Agent 来查
- MCP server：Agent 直接调用工具推荐能力
- 版权声明 + robots.txt 控制大模型爬取
- API 按次计费

## 产品演进路线（三个阶段）

### 阶段1：单工具推荐（现在所有竞品都在这里）
- 「我要做X」→ 推荐一个工具
- TAAFT/Toolify/Futurepedia 都只做到这一步

### 阶段2：多工具组合 Stack（我们已经在做）
- 「我要做X」→ 推荐一套工具组合
- 有 AI Builder，但还是技术人视角

### 阶段3：流程级解决方案（目标方向）
- 「我要改造这个业务流程」→ 推荐完整工作流方案
- 从管理者/企业视角出发
- 每个环节谁做、用什么工具、怎么串起来
- 输出：流程图 + 工作流故事 + 成本估算 + 替代多少人工
- 这不是「工具目录」，是「AI 转型咨询 SaaS」
- 竞品都没有做到这一步

## 用户系统需求

> 从「目录站」变成「服务站」后，用户系统变成刚需。

### 现状
- 没有用户注册/登录系统
- Save Stack 保存到全局 stacks 表（无 user_id）
- 对话历史存 localStorage（浏览器本地，换设备丢失）
- save-stack API 无鉴权

### 轻量方案（推荐先做）
- 保存 stack 时生成唯一 token
- URL 即凭证（/stack/abc123?token=xyz）
- 不需要注册登录，类似 Excalidraw 模式

### 完整方案（有用户量后）
- 加 Clerk（Vercel Marketplace 一键装）
- 数据库加 user_id 字段
- 「我的 Stacks」页面

## SEO 现状评估

### ✅ 已有
- sitemap.ts（动态，含工具/分类/stack/对比页）
- robots.ts（允许爬虫 + 指向 sitemap）
- generateMetadata（工具/stack/分类/对比页都有独立 meta）
- 对比页 /compare/[slugs]（programmatic SEO 已存在）

### ❌ 还缺
- 结构化数据（JSON-LD）— 富摘要
- OG 图片 — 社交分享预览
- canonical URL — 防重复内容
- 内链策略 — 页面之间互相链接
- Google Search Console 提交确认
